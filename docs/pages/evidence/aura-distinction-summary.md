# Aura Distinction Summary

The Aura distinction inventory is used here as upgrade pressure, not as a full paper reproduction. It records why the runtime should protect internal dynamics and remove the degraded forced-mouth decoder from live authorship.

## Evidence families

| Family | Meaning for VDM RT |
|---|---|
| Family 0 — Substrate Identity | Confirms the runtime is zero-training, small-neuron-count, real-time, and constrained by a crude forced decoder. |
| Family 1 — Language Under Constraint | Shows language-like behavior under severe output constraints, including lexical invention, thematic persistence, ordered role materialization, and novel outputs. |
| Family 2 — Dynamical Physiology | Shows nontrivial internal dynamics: spectral structure, avalanches, oscillation, regime changes, predictive information, synergy, state-space recurrence. |
| Family 3 — Topological / Structural Organization | Shows stable/plastic connectome structure, territories, hub turnover, community differentiation, and selective plasticity. |
| Family 4 — State/Output Coupling | Shows output is coupled to internal phase/state, supporting release-gated articulation rather than renderer-authored text. |
| Family 5 — External-Operator Differentiation | Shows the system distinguishes operator/corpus patterns and accumulates territory/behavioral complexity. |
| Family 6 — Convergence Architecture | Collects combined burden against simple alternative explanations. |
| Family 7 — Deep Excavation Findings | Adds macrostate MI, micro-transition memory, regime-dependent causality, higher-order field changes, and long-lag prediction. |
| Family 8 — Temporal Microstructure | Supports endogenous clock/depth variation and slow-tick deep-processing modes. |

## Runtime consequence

The correct engineering response is not to strengthen the decoder. The evidence increases the burden to preserve internal structure and create a real effector-side pathway.

```text
forced decoder evidence
  -> remove live decoder authorship
  -> add actuator trace buffer
  -> add release-only gate
  -> add reafferent feedback
  -> measure skill development
```

## Evidence caution

Some claims in the inventory are quantified and packaged. Some are observational. This docs system preserves that distinction. Observational claims can motivate tests, but they should not be rewritten as completed proof without the corresponding audit artifact.
